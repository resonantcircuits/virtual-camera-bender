How Virtual Camera Bender Works

The Signal Path

Camera preset → Macro dials → Circuit modules → Image.

Cameras are starting points, not limits. Clicking one copies its full settings into the editor. From there everything is yours: every slider, randomizer, and module works from the current state, no matter which camera you started with.

The Gallery (button above the Effects list, or G) shows every camera rendered live on your loaded image, side by side. Click a card to load that camera.

Macros vs Circuit Panels

Macro dials are the big knobs. Moving one re-tunes the module parameters underneath it — so they overwrite fine tweaks in that area. Use them to rough in a look.

Physics Rail is different in kind: it simulates real bend circuitry acting on the raw sensor signal (oscillators into the analog path, logic chips on the ADC data bus). It runs before everything else, and macros never touch it — it changes through its own controls, presets, the Physics randomize family, its dice button, and the physics-led share of Global rolls.

Stylized Circuit is the per-module truth for the plausibility effects: every parameter of every effect. Use it to fine-tune after the macros get you close. Hover a parameter for a short description of what it changes.

Classic Edit is ordinary photo editing after the camera damage is complete: brightness, contrast, saturation, color temperature, and tone cleanup. Macros and random families leave it alone.

Each module header has six controls: the lock freezes that module during right-side randomize, the dice re-rolls just that module, R resets it to defaults, S solos it so you see only its effect, i opens short and long effect notes, and the lamp switches it on or off (there is no separate Enabled toggle inside the group).

ALL OFF switches off every module in a panel. RESET ALL restores every module in that panel to defaults.

Macro Bindings

Bend pushes broad digital stress: exposure gain, false-color saturation, edge burn, vertical smear response, and cheap-camera sharpening.

Color Fault drives false color, contour rings, edge halos, channel damage, and colored sensor noise.

Melt drives vertical smear, pixel sorting, striping, curtain streaks, and some sync wobble.

Burn drives exposure clipping, black crush, contour bands, contour rings, edge burn, and smear trail length.

Noise drives sensor noise, shadow noise, hot pixels, striping, and cheap-camera dither.

Cheapness drives internal downscale, bit depth, dither, sharpening, JPEG/DCT crunch, and some sensor noise.

Chaos drives memory faults, sync tears, buffer ghosting, chroma shift, DCT scramble, and instability mixed into several other modules.

Randomize

Randomize always bends the current settings — it never resets to a preset.

Mode sets how hard it hits: Bent is gentle, Damaged is clearly broken, Shorted is violent, Explore draws from the full range. The per-module dice use this mode too.

Global builds a whole new camera (new name, new seed, fresh macros, and a fresh pick of which modules are active). Some builds are led by the physics rail with the stylized damage pulled back; a few stack both circuits. The family buttons (Physics, Color, Melt, Burn, Noise, Cheap, Memory) re-roll only that damage domain and leave the rest alone — good for keeping a look you like while re-rolling the part you don't.

Locked modules in the right-side panels are restored after right-side randomize, including whether they are enabled. Pressing a locked module's dice does nothing and flashes the lock. Locks are session-only and are not saved into presets.

Seed

The seed steers every random pattern in the render: noise grain, drip positions, block glitches. Reroll keeps all your settings but redraws the damage in new places. Same settings + same seed = same image.

The seed lock only affects right-side randomize. Manual Reroll still changes the seed on purpose. Seed lock is session-only and is not saved into presets.

Comparing

Split A/B overlays the original left of a divider you drag across the image. Hold · Original (or holding C) flashes the untouched photo.

Mouse wheel zooms the preview image. When zoomed, drag the image to pan and double-click to reset the view.

Export

Format chooses PNG, WebP, or JPEG. Size chooses what gets written.

Preview exports the same scaled render shown in the UI, capped at 1500 px on the longest edge. This is the default because resolution-dependent effects like contour rings, noise, sorting, and OSD match what you see.

Full re-renders the original source resolution. It can take longer, make larger files, and look denser because many effects are intentionally resolution-dependent.

Supported browsers open a save dialog for filename and location. Others ask for a filename and then use the browser's normal download folder.

Video

Loading a video opens the Frames contact sheet (V): 25 evenly spaced frames rendered through the current camera. Click a frame to make it the working image — every editor tool then works on that frame exactly as it will render in the final video.

The Temporal panel controls behavior across frames. Seed Mode: locked keeps all damage frozen in place, hold re-rolls it every ~N frames (camcorder stutter), flicker re-rolls every frame. Sensor grain and speckle always shimmer per frame regardless — only the structural damage obeys the mode. Drift makes module parameters wander slowly so the look evolves. Ghost Lag feeds the Buffer Ghost module the frame N back — real stale-buffer trails (enable Buffer Ghost to see it).

The app never encodes video. Copy Render Command gives you an OS-formatted CLI line with the current camera embedded, so no preset file is needed. Choose Smaller File, Balanced, or High Quality before copying, then run it with ffmpeg installed.

Presets and Session State

Save Preset writes camera settings, seed, macros, module parameters, output format, and export size. It does not save the source photo, ghost source image, module locks, seed lock, undo history, or split/solo view state.

Buffer Ghost can use a loaded second image as the stale frame. If no ghost image is loaded, it falls back to the current image.
